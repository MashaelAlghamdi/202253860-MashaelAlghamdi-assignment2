# AI Usage Report

## 1. AI Tools Used

The following AI tools were used during the development of this project:

- **ChatGPT**
- **GitHub Copilot**

These tools were used as supporting resources to assist with debugging, understanding concepts, and improving code organization.

---

## 2. How AI Was Used (Use Cases)

AI was used during several stages of development to support problem solving and code refinement.

The tools were used in the following ways:

- **ChatGPT** – Used to clarify JavaScript logic, particularly for implementing dark mode using `localStorage`, understanding CSS specificity, and improving the structure and readability of the code. It was also helpful when debugging certain interactive features such as project expansion behavior and form validation.

- **GitHub Copilot** – Used inside the code editor to suggest syntax improvements and complete repetitive coding patterns. Copilot was especially useful for small JavaScript functions and CSS rules.

ChatGPT was mainly used for explanations and debugging discussions, while GitHub Copilot assisted with real-time code suggestions during development.

---

## 3. Benefits of Using AI

Using AI tools helped improve both development speed and understanding of certain technical concepts.

Some key benefits included:

- Faster debugging and problem-solving.
- Better understanding of CSS behavior, particularly specificity and style overrides.
- Learning how to implement dark mode using `localStorage`.
- Assistance with improving the readability and structure of code and documentation.

AI also helped provide suggestions for improving the user interface and interaction design of the portfolio.

---

## 4. Challenges and Limitations

Some AI-generated suggestions required adjustment before they could be used in the project.

For example:

- Certain styling suggestions needed modification to match the visual theme of the portfolio.
- Some code examples had to be adapted to fit the existing HTML structure.

All suggestions were tested and modified when necessary to ensure they worked correctly within the project.

---

## 5. Learning Outcomes

Through using AI tools during this project, several concepts became clearer.

Some of the key learning outcomes include:

- Understanding how to implement a theme toggle using JavaScript by dynamically adding and removing a CSS class from the `<body>` element.
- Learning how `localStorage` can store user preferences so that the selected theme remains saved even after refreshing the page.
- Understanding how CSS specificity allows dark mode styles (such as `body.dark .navbar`) to override default styles.
- Practicing how to structure responsive layouts using Flexbox and media queries.
- Implementing interactive features such as project searching, sorting, expandable project cards, and form validation.

These experiences helped strengthen both front-end development skills and problem-solving approaches.

---

## 6. Responsible Use and Modifications

All AI-generated suggestions were carefully reviewed before being integrated into the project.

Any code suggested by AI tools was tested locally and modified where necessary to ensure that it worked correctly and matched the intended functionality.

The final implementation reflects personal understanding of the code and its behavior.

AI was used as a learning assistant rather than a direct replacement for the development process.

---

## 7. Innovative Use of AI

AI was used not only for generating code suggestions but also as a decision-support tool during development.

Different layout ideas and styling approaches were explored based on AI suggestions. These ideas were then tested and refined before selecting the final design.

AI also helped streamline debugging and concept clarification, allowing more time to focus on improving user experience, responsiveness, and overall project structure.
